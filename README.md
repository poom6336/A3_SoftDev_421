# Oscilloscope SCPI Controller

## Description
The **Oscilloscope SCPI Controller** is a lightweight Python-based Graphical User Interface (GUI) designed to interface with digital oscilloscopes. Using the VISA standard via `pyvisa`, it allows users to discover connected instruments, send Standard Commands for Programmable Instruments (SCPI), toggle channel settings, and view a live stream of the oscilloscope's display. It also features a built-in auto-repair mechanism for common PNG checksum (CRC) errors generated by certain oscilloscope firmwares, ensuring smooth rendering in Tkinter.

## Objectives
*   **Remote Control:** Provide an easy-to-use interface to interact with oscilloscopes over USB, GPIB, or TCP/IP.
*   **Command Testing:** Offer a direct console to test raw SCPI commands and queries with clear logging.
*   **Live Monitoring:** Stream and display the oscilloscope's screen in near real-time.
*   **Data Capture:** Allow users to instantly save high-quality PNG screenshots of their waveforms directly to their local machine.
*   **Accessibility:** Use a purely Python-based backend (`pyvisa-py`) to minimize heavy proprietary driver installations (like NI-VISA) where possible.

---

## Required Software Packages

To run this application, you will need **Python 3.7+** and the following packages:

*   **`pyvisa`**: The core library for interacting with VISA instruments.
*   **`pyvisa-py`**: The pure Python backend for PyVISA (required since the app defaults to the `@py` backend).
*   **`tkinter`**: Python's standard GUI package (usually included with standard Python installations, but may require manual installation on some Linux distributions).

---

## Installation Instructions

### 1. Install Python
Ensure Python 3 is installed on your system. You can download it from [python.org](https://www.python.org/).

### 2. Install Required Python Libraries
Open your terminal or command prompt and run the following command to install the PyVISA dependencies:

```bash
pip install pyvisa pyvisa-py
```

*Note for Linux Users:* If `tkinter` is not installed by default on your distribution, you may need to install it via your package manager. For example, on Ubuntu/Debian:
```bash
sudo apt-get install python3-tk
```

### 3. Hardware Drivers (Optional but Recommended)
While the `@py` backend is highly capable, USB instruments on Windows might require a libusb backend or a standard VISA implementation. If you have issues connecting, you may need to install [NI-VISA](https://www.ni.com/en/support/downloads/drivers/download.ni-visa.html) or [Keysight IO Libraries](https://www.keysight.com/us/en/lib/software-detail/computer-software/io-libraries-suite-downloads-2175637.html) and change the backend in the code from `backend="@py"` to `backend=""`.

## Usage

### 1. Launch the Application
Save the provided Python code as a file (e.g., `osci_controller.py`) and run it from your terminal:

```bash
python osci_controller.py
```

### 2. Connect to Your Instrument
1. Ensure your oscilloscope is powered on and connected to your PC via USB, LAN, or GPIB.
2. Click the **Connect** button in the top left corner.
3. The application will scan for valid resources. If successful, the instrument's identification string (`*IDN?`) and connection status will turn green.

### 3. Sending SCPI Commands
*   Type any valid SCPI command into the text entry field (e.g., `:TIM:SCAL 0.001`).
*   Press **Enter** or click **Send**.
*   **Tip:** If your command contains a question mark `?` (e.g., `*IDN?` or `:MEAS:VPP?`), the application will automatically treat it as a query, read the response from the scope, and display it in the log window.

### 4. Quick Channel Controls
Use the provided grid on the left side of the application to quickly manage Channels 1 through 4.
*   Check **Enable Channel** to toggle a channel's trace on or off.
*   Select a voltage scale from the dropdown and click **Set** to adjust the Volts/Division.

### 5. Live Display & Saving Captures
*   Once connected, the right panel will automatically begin streaming screenshots from the oscilloscope at approximately 1 frame per second.
*   To save a waveform for documentation, click the **Save Capture** button in the top right corner of the display panel. A file dialog will prompt you to save the high-resolution `.png` file to your computer.
